关于CSS预处理与经典布局

CSS预处理器

  • 基于CSS的另一种语言
  • 通过工具编译为CSS
  • 添加了很多CSS不具备的特性
  • 主要有less和sass

输出CSS：

mixin

如果有一段CSS样式需要复用，没有预处理器的做法是给需要这个样式的元素加上相同的class，也就是在HTML中复用。

如果想在CSS中复用而不增加class可以用mixin（类似一个函数？）：

@fontSize: 12px;
@bgColor: red;

.block(@fontSize) {
    font-size: @fontSize;
    border: 1px solid #ccc;
    border-radius: 4px;
}

.wrapper {
    background-color: aliceblue;
    .nav {
        .block(@fontSize);
    }
    .content {
        .block(@fontSize + 2px);
        &:hover {
            background-color: lighten(@bgColor, 40%);
        }
    }
}

// CSS
.wrapper {
  background-color: aliceblue;
}
.wrapper .nav {
  font-size: 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
}
.wrapper .content {
  font-size: 14px;
  border: 1px solid #ccc;
  border-radius: 4px;
}
.wrapper .content:hover {
  background-color: #ffcccc;
}

注意，不加括号的类也可以作为mixin，但加上括号后不会在CSS中编译出来（空括号也可以）

extend

太多的mixin也会有很多重复代码，可以用extend复用，它能把公共样式提取后整合选择器

@fontSize: 12px;
@bgColor: red;

.block{
    font-size: @fontSize;
    border: 1px solid #ccc;
    border-radius: 4px;
}

.wrapper {
    background-color: aliceblue;
    .nav:extend(.block){
        color: #333;
    }
    .content {
        &:extend(.block);
        &:hover {
            background-color: lighten(@bgColor, 40%);
        }
    }
}

// CSS
.block,
.wrapper .nav,
.wrapper .content {
  font-size: 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
}
.wrapper {
  background-color: aliceblue;
}
.wrapper .nav {
  color: #333;
}
.wrapper .content:hover {
  background-color: #ffcccc;
}

loop

用mixin能传参的性质实现，没有循环，实际是用递归实现的：

.gen-col(@n) when (@n > 0) {
    .gen-col(@n - 1);
    .col-@{n} {
        width: 100px / 5 * @n;
    }
}
.gen-col(5);

// CSS
.col-1 {
  width: 100px / 5 * 1;
}
.col-2 {
  width: 100px / 5 * 2;
}
.col-3 {
  width: 100px / 5 * 3;
}
.col-4 {
  width: 100px / 5 * 4;
}
.col-5 {
  width: 100px / 5 * 5;
}

import CSS模块化

将不同部分的CSS拆分编写，之后统一import进一个文件后编译就能编译到一个大的总集合CSS文件中：

// variable.less
@themeColor: blue;
@fontSize: 14px;

// module1.less
.module1 {
    .box {
        font-size: @fontSize + 2px;
        color: @themeColor;
    }
    .tips {
        font-size: @fontSize;
        color: lighten(@themeColor, 40%);
    }
}

// module2.less
.module2 {
    .box {
        font-size: @fontSize + 4px;
        color: @themeColor;
    }
    .tips {
        font-size: @fontSize + 2px;
        color: lighten(@themeColor, 20%);
    }
}

// index.less 依然只用lessc编译这个文件
@import "./variable.less";
@import "./module1.less";
@import "./module2.less";

// CSS
.module1 .box {
  font-size: 16px;
  color: blue;
}
.module1 .tips {
  font-size: 14px;
  color: #ccccff;
}
.module2 .box {
  font-size: 18px;
  color: blue;
}
.module2 .tips {
  font-size: 16px;
  color: #6666ff;
}

三栏布局

三栏布局一般指的是页面中一共有三栏，左右两栏宽度固定，中间自适应的布局，注意为了保证中间的内容最先被渲染，center的盒子是要放在最上面的！html代码如下：

<div class="container">
    <div class="center"></div>
    <div class="left"></div>
    <div class="right"></div>
</div>

以下样式全部按照左边定宽100px，中间自适应，右边定宽200px

绝对定位实现

左右两栏设置绝对定位，中间设置对应的margin

.container {
    position: relative;
}
.left{
    background-color: coral;
    position: absolute;
    width: 100px;
    height: 100px;
    top: 0;
    left: 0;
}
.right {
    background-color: cornflowerblue;
    position: absolute;
    width: 200px;
    height: 100px;
    top: 0;
    right: 0;
}
.center {
    background-color: darkgreen;
    height: 100px;
    margin: 0 200px 0 100px;
}

flex实现

左右两栏的放大和缩小比例都设置为0，基础大小设置为固定的大小，中间一栏设置为auto

.container {
    display: flex;
}
.left{
    background-color: coral;
    width: 100px;
    height: 100px;
    flex: 0 0 100px;
    order: 0;
}
.right {
    background-color: cornflowerblue;
    width: 200px;
    height: 100px;
    flex: 0 0 200px;
    order: 2;
}
.center {
    background-color: darkgreen;
    height: 100px;
    flex: 1 0 auto;
    order: 1;
}

记得要加order指定下排列顺序

圣杯布局

利用浮动和负边距来实现。父级元素设置左右的 padding，三列均设置浮动，两列都被挤到了下一行，通过设置 margin 负值将其移动到上一行，再利用相对定位到两边。

.container {
    padding-left: 100px;
    padding-right: 200px;
}
.left{
    background-color: coral;
    width: 100px;
    height: 100px;
    float: left;
    position: relative;
    left: -100px;
    margin-left: -100%;
}
.right {
    background-color: cornflowerblue;
    width: 200px;
    height: 100px;
    float: right;
    margin-right: -200px;
}
.center {
    background-color: darkgreen;
    width: 100%;
    height: 100px;
    float: left;
}

宽高自适应的正方形

html如下：

<div class="square">
</div>

用vw实现

什么是vw？

​ 响应式布局的实现依靠媒体查询（ Media Queries ）来实现，选取主流设备宽度尺寸作为断点针对性写额外的样式进行适配，但这样做会比较麻烦，只能在选取的几个主流设备尺寸下呈现完美适配。 即使是通过 rem 单位来实现适配，也是需要内嵌一段脚本去动态计算根元素大小。

​ 在桌面端，视口指的是浏览器的可视区域；而在移动端较为复杂，它涉及到三个视口：分别是 Layout Viewport（布局视口）、 Visual Viewport（视觉视口）、Ideal Viewport。 而视口单位中的“视口”，在桌面端，毫无疑问指的就是浏览器的可视区域；但是在移动端，它指的则是三个 Viewport 中的 Layout Viewport 。

根据CSS3规范，视口单位主要包括以下4个：

  • vw : 1vw 等于视口宽度的1%
  • vh : 1vh 等于视口高度的1%
  • vmin : 选取 vw 和 vh 中最小的那个
  • vmax : 选取 vw 和 vh 中最大的那个

注意：视口单位区别于%单位，视口单位是依赖于视口的尺寸，根据视口尺寸的百分比来定义的；而%单位则是依赖于元素的祖先元素。

CSS实现：

.square {
    background-color: #bfa;
    width: 10vw;
    height: 10vw;
}

margin/padding实现

元素的margin/padding在设置百分比时是相对父元素width的：

.square {
    background-color: #bfa;
    width: 10vw;
    padding-top: 10%;
}

利用子元素的margin-top

给这个盒子加个儿子伪元素，而这个儿子的margin百分比也是按它本身宽度来的，注意父元素要开启BFC，不然儿子的MT会加在父元素头上而不是内部，也就撑不开父元素：

.square {
    background-color: #bfa;
    overflow: hidden;
    width: 30vw;
}
.square::after {
    content: '';
    display: block;
    margin-top: 100%;
}

三角形

利用元素边框连接处会等分的原理：（画圆用border-radius）

比如一个角朝上的三角形，要求底宽100px，高50px（延用上文的square）：

.square {
    width: 0;
    border-left: 50px solid transparent;
    border-right: 50px solid transparent;
    border-bottom: 50px solid #bfa;
}
