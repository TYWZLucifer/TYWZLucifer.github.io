变量类型和计算

值类型和引用类型

值类型用栈存储，引用类型用堆存储，不同存储的原因是数据结构的不同，值类型占内存小且结构简单，引用类型相反。

常见引用类型：

const obj = {};
const arr = [1,2,3];
const n = null; // 特殊引用类型 指针指向空
function fn() {} // 特殊引用类型 不用于存储数据所以没有“深拷贝”

typeof和深拷贝

typeof：

  • 识别所有值类型
  • 能判断函数 ‘function’
  • 识别引用类型 ‘object’ 但无法继续识别更深层次，比如这个对象属于哪类 –> null和{x:100}都是‘object’

深拷贝：记得递归检查对象内部的值是否也是复杂对象

function deepClone(obj = {}) {
    if(typeof obj !== 'object' || obj == null) return obj;
    let res;
    if(obj instanceof Array) res = [];
    else res = {};
    for(let key in obj) {
        // 保证key不是原型属性
        if(obj.hasOwnProperty(key)) {
            res[key] = deepClone(obj[key]);
        }
    }
    return res;
}

function inheritPrototype(subType, superType) {
    function F() {};
    F.prototype = superType.prototype;

instanceof可以判断引用类型的数据是否是由某个类实例化的

终点是一个object的实例，__proto__为空

进阶内容：

1.object对象是由 object()方法实例化的，而 object()函数也是对象，是由 function object()声明，这个函数对象的 prototype指向上文中所说的终点（__proto__为空的 object实例），而其自身还拥有一个__proto__，指向构造它的函数的 prototype，也就是 function的 prototype。

2.function.prototype是一个 object实例对象，其__proto__指向上文的终点。而 function也是由 function声明的（没错自己是自己的构造函数），所以它是唯一自身的__proto__指向 prototype的函数，对于其他函数而言，由于都是 funciton构造的，所以所有函数的__proto__都指向 function()的 prototype

作用域和闭包

闭包

ES6新增了块级作用域

闭包：作用域应用的特殊情况，一般有两种表现

  • 函数作为参数被传递
  • 函数作为返回值被返回

自由变量的查找是从函数定义的地方向上层作用域查找，而不是在执行的作用域查找

// 函数作为返回值
function create() {
    let a = 100;
    return function () {
        console.log(a);
    }
}
const fn1 = create();
const a = 200;
fn1(); // 100

// 函数作为参数
function printm(fn) {
    const b = 200;
    fn();
}
const b = 100;
function fn2() {
    console.log(b);
}
printm(fn2); // 100

闭包的作用：

可以隐藏数据，只提供API，防止手动修改数据

function createCache() {
    const data = {}; // 将data隐藏，外界无法通过data.a更改数据
    return {
        set(key, value) {
            data[key] = value;
        },
        get(key) {
            return data[key];
        }
    };
}
let x = createCache();
x.set('a',100);
console.log(x.get('a'))

建立不同的隐藏作用域

for(var i = 0; i < 10; i++) {
    a[i].onclick = function() {
        alert(i);
    }
} // 每个a点击后都出现 10

for(var i = 0; i < 10; i++) {
    (function(num) {
        a[num].onclick = function() {
            alert(num);
        }

// 但是let与var不同在于let有块级作用域（每个循环一个块作用域），每个a都能输出对应的数字

}

this

this的取值是在函数执行时判断，而不是定义时，适用于所有普通情况=>普通函数、call、apply、bind、对象方法、class

而对于箭头函数，this是在定义函数时绑定的，而不是执行过程中。（这样做解决了匿名函数、setTimeout的this指向问题），当然用上文闭包中变量在函数定义的地方查找的原理，可以先用一个that保存this，使用that就不会有这些问题了。。

const a = {
    name: 'a',
    say() {
        console.log(this); // a
    },
    wait() {
        setTimeout(function() {
            console.log(this); // window
        })
    }
}

const b = {
    name: 'b',
    say() {
        console.log(this); // b
    },
    wait() {
        setTimeout(() => {
            console.log(this); // b
        })
    }
}

改变this的方式：bind，call，apply。

某一对象.调用方法.call(准备使用方法的对象, 参数)

1.判断调用对象是否为函数，即使我们是定义在函数的原型上的，但是可能出现使用 call 等方式调用的情况。

2.判断传入上下文对象是否存在，如果不存在，则设置为 window 。

3.处理传入的参数，截取第一个参数后的所有参数。

4.将函数作为上下文对象的一个属性。

5.使用上下文对象来调用这个方法，并保存返回结果。

6.删除刚才新增的属性。

7.返回结果。

apply和bind情况类似，不同点在于，apply传入参数为一整个数组，不需要解构。而bind需要判断最终返回的函数是否会作为一个构造函数调用，如果作为构造函数使用，构造的对象this就是执行上下文，而不是传入的那个context。

function call(context) {
    if(typeof this !== "function") console.error("type error");
    context = context || window;
    context.fn = this;
    let args = [...arguments].slice(1), res;
    res = context.fn(...args);
    delete context.fn;
    return res;
}
function apply(context) {
    if(typeof this !== "function") console.error("type error");
    context = context || window;
    context.fn = this;
    let res = null;
    if(arguments[1]) res = context.fn(...arguments[1]);
    else context.fn();
    delete context.fn;
    return res;
}
function bind(context) {
    if(typeof this !== "function") console.error("type error");
    let args = [...arguments].slice(1), fn = this;
    return function Fn() {
        return fn.apply(
            this instanceof Fn ? this : context,
            args.concat(...arguments)
        );
    }
}

实现关键字new（new中也有this的问题）

1.首先创建了一个新的空对象

2.设置原型，将对象的原型设置为函数的 prototype 对象。

3.让函数的 this 指向这个对象，执行构造函数的代码（为这个新对象添加属性）

4.如果构造器没有手动返回对象，则返回第一步创建的新对象，如果有，则舍弃掉第一步创建的新对象，返回手动return的对象。

function objectFactory() {
    let newObj = null, constructor = [].shift.call(arguments);
    if(typeof constructor !== "function") console.error("type error");
    newObj = Object.create(constructor.prototype);
    let res = constructor.apply(newObj, [...arguments]);
    return typeof res === "object" ? res : newObj;
}

事件

event.preventDefault() 阻止默认行为，比如超链接的跳转等 （当a标签的href属性存在并有一个正确的地址时，点击它，它会跳转到对应的地址去，这就是它的默认行为）

DOM0与DOM2级事件的区别：

  • DOM0绑定多个事件后只执行最后一个，而DOM2可以绑定多个，只有当事件名，函数名，发生阶段都一样才会只执行最后一个
  • DOM0不需要考虑兼容问题
  • DOM0事件规定的事件流包含3个阶段，事件捕获阶段、处于目标阶段、事件冒泡阶段。首先发生的事件捕获为截获事件提供机会，然后是实际的目标接收事件，最后一个阶段是事件冒泡阶段，可以在这个阶段对事件做出响应。也就是说，DOM0级事件只能触发事件冒泡阶段而不能触发事件捕获阶段
  • DOM2级事件使用addEventListener和removeEventListener创建和删除，第三个参数为true时代表在捕获期间触发

阻止冒泡使用：

  • event.stopPropagation()，可以在DOM0，DOM2使用
  • event.target==this 判断是否是当前元素触发的绑定事件

事件代理：利用冒泡机制，给父元素绑定监听事件

  • 可以大量节省内存占用，减少事件注册
  • 实现当新增子对象时无需再次对其绑定
// 通用的事件监听函数 包括普通绑定和代理绑定
function bindEvent(elem, type, selector, fn) {
    if(fn == null) [fn, selector] = [selector, fn]; // 普通绑定第三个参数为fn
    elem.addEventListener(type, event => {
        const target = event.target;
        if(selector) { // 代理绑定
            if(target.matches(selector)) fn.call(target, event);
            else fn.call(target, event);
        }
    })
}
const ula = document.getElementById('thisUl');
bindEvent(ula, 'click', 'li', function(e) {e.target.innerHTML});

Ajax

XMLHttpRequest封装一个简单的ajax

function ajax(url, sucessFn) {
    const xhr = new XMLHttpRequest();
    xhr.open('GET', url, true); // true为 异步请求
    xhr.send(null);
    xhr.onreadystatechange = function () {
        if(xhr.readyState === 4) {
            if(xhr.status === 200) {
                sucessFn(xhr.responseText);
            }
        }
    }
}
// post 请求,替换send内容，必须是一个字符串
const data = {
    value: 123
}
// xhr.send(JSON.stringify(data))

跨域：ajax请求时，浏览器要求当前网页和server必须同源（安全）

  • 同源：协议、域名、端口三者一致
  • 加载图片、css、js可以无视同源策略<img src=imgurl />、<link href=cssurl />、<script src=jsurl></script>，也就是说CDN引入可以跨域 tips：<img>可以用来统计打点，访问静态页面时,通过img标签指定src 为访问统计的地址, img标签向统计程序发出请求,实现统计，也可以使用第三方服务，如百度的提供的统计等

解决方式：

  • JSONP：利用<script>可以绕过跨域限制，服务器可以任意拼接数据返回

    <script>
    window.callback = function(data) {
        console.log(data);
    }
</script>
<script src="http://xxx.xxx/jsonp.js?username=xxx"></script> // 服务器可以拿到username

// jsonp.js
callback(
	{ name: 'dfg' }
)

    请求的数据会立马被浏览器当作javascript语句去执行，所以js文件中用一个函数调用数据的形式，变成一个可执行的js代码，也就是上文的callback(data)，这个函数定义在浏览器端的代码中，这样想要的数据就在包裹中完成传递

  • CORS：全称是”跨域资源共享”（Cross-origin resource sharing），需要浏览器和服务器同时支持。目前，所有浏览器都支持该功能，IE浏览器不能低于IE10。浏览器会在ajax简单请求中添加一个origin字段，用于指出当前请求属于哪个域（协议、域名、端口），之后服务器根据这个值决定是否允许其跨域，如果允许，返回的响应头需要携带以下信息：

    Access-Control-Allow-Origin: http://manage.leyou.com // 可接受的域，是一个具体域名或者*，代表任意
Access-Control-Allow-Credentials: true // 是否允许携带cookie，默认情况下，cors不会携带cookie，除非这个值是true
Content-Type: text/html; charset=utf-8

    如果是特殊请求，比如PUT之类的，会在正式通信前用option发一次预检请求，增加信息如下：

    Access-Control-Request-Method：接下来会用到的请求方式，比如PUT
Access-Control-Request-Headers：会额外用到的头信息

// 例子
OPTIONS /cors HTTP/1.1
Origin: http://manage.leyou.com
Access-Control-Request-Method: PUT
Access-Control-Request-Headers: X-Custom-Header
Host: api.leyou.com
Accept-Language: en-US
Connection: keep-alive
User-Agent: Mozilla/5.0...

    server端收到option预检请求，如果允许跨域，会发出响应，包含以下增加信息：

    Access-Control-Allow-Methods：允许访问的方式
Access-Control-Allow-Headers：允许携带的头
Access-Control-Max-Age：本次许可的有效时长，单位是秒，过期之前的ajax请求就无需再次进行预检了

存储

cookie：

  • 本身用于浏览器与server通讯（请求头），被“借用”到本地存储（HTML5之前）

  • 用document.cookie='...'的方式来修改，且只能这样修改

  • 字符串形式，key=value，用；分割，有以下几个属性：

    name // cookie的名称
value // cookie的值
domain // 可以访问此cookie的域名
/*
	设置：
	非顶级域名，如二级域名或者三级域名，设置的cookie的domain只能为顶级域名或者二级域名或者三级域名本身，不能设置其他二级域名的cookie，否则cookie无法生成。
	顶级域名只能设置domain为顶级域名，不能设置为二级域名或者三级域名，否则cookie无法生成。
	
	取值：
	二级域名能读取设置了domain为顶级域名或者自身的cookie，不能读取其他二级域名domain的cookie。所以要想cookie在多个二级域名中共享，需要设置domain为顶级域名，这样就可以在所有二级域名里面或者到这个cookie的值了。
	顶级域名只能获取到domain设置为顶级域名的cookie，其他domain设置为二级域名的无法获取。
*/
path // domain下可访问此cookie的路径
expires/Max-Age // 过期时间
/*
	此cookie超时时间。若设置其值为一个时间，那么当到达此时间后，此cookie失效。不设置的话默认值是Session，意思是cookie会和session一起失效。当浏览器关闭(不是浏览器标签页，而是整个浏览器) 后，此cookie失效。
*/
Size // cookie大小
http 
/*
	cookie的httponly属性。若此属性为true，则只有在http请求头中会带有此cookie的信息，而不能通过document.cookie来访问此cookie
*/
secure // 设置是否只能通过https来传递此条cookie

  • 存储大小4KB（单个cookie），且http请求会将它发送到服务端，增加请求的数据量

localStorage和sessionStorage：

  • HTML5中专门为存储设计，最大5m（针对每个域名！！）

  • API用setItem、getItem

    localStorage.setItem('a', 100);
localStorage.getItem('a') // "100" 是字符串形式
localStorage.removeItem('a');

sessionStorage.setItem('a', 100);
sessionStorage.getItem('a') // "100" 是字符串形式
sessionStorage.removeItem('a');

  • 不会随着http请求发送出去

  • localStorage数据会永久存储，除非代码或手动删除，而sessionStorage只存在于当前会话，浏览器关闭会清空
