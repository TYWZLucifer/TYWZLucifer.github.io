判断二分图

看了题解，其实就是遍历图，用染色法在遍历的过程中，将相邻的节点分成不同颜色集合的，如果能分完就返回true，遇到相邻节点但是是一个颜色返回false，DFS\BFS都可以

class Solution {
    private static final int UNCOLORED = 0;
    private static final int RED = 1;
    private static final int GREEN = 2;
    private int[] color;
    private boolean valid;

    public boolean isBipartite(int[][] graph) {
        int n = graph.length;
        color = new int[n];
        valid = true;
        Arrays.fill(color,UNCOLORED);
        for(int i=0;i<n && valid;i++) {
            if(color[i] == UNCOLORED)
                dfs(i,RED,graph);
        }
        return valid;
    }

    public void dfs(int cur, int cur_co, int[][] graph) {
        color[cur] = cur_co;
        int nei_co = color[cur]==RED?GREEN:RED;
        for(int nei:graph[cur]) {
            if(color[nei] == UNCOLORED) {
                dfs(nei,nei_co,graph);
                if(!valid) return;
            }
            else if(color[nei] != nei_co) {
                valid = false;
                return;
            }    
        }
    }
}

滑动窗口的最大值

这尼玛居然是简单难度。。。看K神题解用之前的含有min函数的栈那个方法去做，用队列保存一个最大值，这样每次窗口取最大值的时间复杂度就是O(1)

class Solution {
    public int[] maxSlidingWindow(int[] nums, int k) {
        if(nums.length==0 || k==0) return new int[0];
        Deque<Integer> q = new LinkedList<>();
        int[] res = new int[nums.length-k+1];
        for(int i=0;i<k;i++) {
            while(!q.isEmpty() && q.peekLast()<nums[i]) q.removeLast();
            q.addLast(nums[i]);
        }
        res[0]=q.peekFirst();
        for(int i=0,j=k; j < nums.length; i++,j++) { // 形成窗口后
            if(q.peekFirst() == nums[i]) q.removeFirst();
            while(!q.isEmpty() && q.peekLast() < nums[j]) q.removeLast();
            q.addLast(nums[j]);
            res[i+1] = q.peekFirst();
        }
        return res;
    }
}

class Solution {
    public int searchInsert(int[] nums, int target) {
        int l=0, r=nums.length-1;
        while(l<=r) {
            int mid=l+(r-l)/2;
            if(nums[mid]<target) l=mid+1;
            else if(nums[mid]>target) r=mid-1;
            else return mid; 
        }
        return l;
    }
}

队列的最大值

比较辅助队列和主队列的值是否相等时不能用==,需要用equals，思想和之前栈的那个题一样

class MaxQueue {
    
    Deque<Integer> q;
    Deque<Integer> help_q;

    public MaxQueue() {
        q = new LinkedList<>();
        help_q = new LinkedList<>();
    }
    
    public int max_value() {
        if(q.isEmpty()) return -1;
        return help_q.peekFirst();
    }
    
    public void push_back(int value) {
        q.addLast(value);
        while(!help_q.isEmpty() && help_q.peekLast()<value) help_q.removeLast();
        help_q.addLast(value);
    }
    
    public int pop_front() {
        if(q.isEmpty()) return -1;
        if(q.peekFirst().equals(help_q.peekFirst())) help_q.removeFirst();
        return q.poll();
    }
}

交错字符串

和室友讨论想到动规，类似机器人找路径的二维矩阵，只能往左或往下走

class Solution {
    public boolean isInterleave(String s1, String s2, String s3) {
        int n = s1.length();
        int m = s2.length();

        if(s3.length() != n+m) return false;

        boolean[] dp = new boolean[m+1];
        dp[0] = true;
        for(int i=0;i<=n;i++) {
            for(int j=0;j<=m;j++) {
                int p = i+j-1;
                if(i>0) dp[j] = dp[j]&&s1.charAt(i-1)==s3.charAt(p);
                if(j>0) dp[j] |= dp[j-1]&&s2.charAt(j-1)==s3.charAt(p);
            }
        }

        return dp[m];
    }
}

戳气球

这他妈都能动态规划？原来动规不都是从前往后遍历一次的

class Solution {
    public int maxCoins(int[] nums) {
        int n = nums.length;
        int[][] rec = new int[n+2][n+2];
        int[] val = new int[n+2];
        val[0] = val[n+1] = 1;
        for(int i=1;i<=n;i++) val[i] = nums[i-1];
        for(int i=n-1;i>=0;i--) {
            for(int j=i+2;j<=n+1;j++) {
                for(int k=i+1;k<j;k++) {
                    int sum = val[i]*val[k]*val[j];
                    sum += rec[i][k] + rec[k][j];
                    rec[i][j] = Math.max(rec[i][j],sum);
                }
            }
        }
        return rec[0][n+1];
    }
}

n个骰子的点数

这题居然是简单题。。。最高赞的题解是计算出每种点数出现的次数然后除以总次数，我觉得不如这个老哥直接计算概率

class Solution {
    public double[] twoSum(int n) {
        double pre[]={1/6d,1/6d,1/6d,1/6d,1/6d,1/6d};
        for(int i=2;i<=n;i++) {
            double[] tmp = new double[5*i+1];
            for(int j=0;j<pre.length;j++) {
                for(int x=0;x<6;x++) {
                    tmp[j+x] += pre[j]/6;
                }
            }
            pre = tmp;
        }
        return pre;
    }
}

圆圈中最后剩下的数字

最近的简单题是真滴牛批，约瑟夫环倒推就两行，我想不出来。。

class Solution {
    public int lastRemaining(int n, int m) {
        int ans = 0;
        // 最后一轮剩下2个人，所以从2开始反推
        for (int i = 2; i <= n; i++) {
            ans = (ans + m) % i;
        }
        return ans;
    }
}
